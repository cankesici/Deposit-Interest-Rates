# a) The model we are building is sensible and similar models using CBT/swap rates are used quite extensively in real world banking applications.
# We could also include a combination of variables such as inflation, or USDTRY rate, GDP growth or other macroeconomic variables such as unemployment
# rate or government bond yields. This would make sense in environments where monetary policy transmission mechanisms are less direct or where market  
# conditions deviate significantly from the norm. This would however significantly increase model complexity and may raise issues with model estimation.

# b) The two variables that would be most suitable for modeling deposit rates are CBT and OIS1Y. CBT represents the monetary policy stance set by the central 
# bank, directly impacting deposit rates. OIS1Y reflects short-term interest rate expectations in the interbank market. Including both CBT and OIS1Y in 
# the model captures both policy-driven influences and market-driven expectations. Since the question already advises us that banks use Banks base their 
# deposit pricing on the prevailing interbank market rates, and often use either the Central Bank policy rates or short term swap rates since deposits 
# are short term products, it is enough two use two variables to capture this aforementioned relationship. including too many variables can lead to overfitting, 
# where the model the learns noise in the data rather than true relationships. A simpler model with fewer variables is often easier to interpret and explain 
# to stakeholders. Adding more variables also increases the risk of multicollinearity, where independent variables are highly correlated with each other, 
# which is especially true for the different OIS rates. This can make it difficult to estimate the individual effect of each variable on the dependent 
# variable accurately.

# Time series tests

# c) If some tests fail, it indicates that certain model assumptions may not hold true. In such cases, we can consider techniques such as 
# transforming the data, incorporating additional variables, or using robust regression methods. Additionally, we may need to reevaluate the 
# data preprocessing steps or collect more data if necessary.

# Linear regression assumptions: linearity, independence, homoscedasticity, normality of residuals, no perfect multicollinearity
# Time series assumptions: Stationarity (constant properties), independence of errors, autocorrelation (conflicting with lin. reg.), etc.

# Assuming the p-value for the confidence level is 0.05 for all tests
# Residuals calculation
residuals = y_train - y_pred

## Stationarity Test (ADF)
# Values below 0.05 suggest non-stationartiy
# Non-stationarity can be addressed by differencing the data or applying transformations such as log transformation
adf_test = sm.tsa.adfuller(residuals)
print('ADF test p-value', adf_test[1])

# Autocorrelation Analysis (Durbin-Watson test)
# Values close to 2 indicate no autocorrelation, while deviations suggest autocorrelation. 
# Autocorrelation can be addressed by adding lagged variables to the model or using time series techniques like ARIMA.
durbin_watson_statistic = durbin_watson(residuals)
print("Durbin-Watson test statistic", durbin_watson_statistic)

# White's Heteroskedasticity Test
# White's test p-value less than 0.05 suggests heteroskedasticity. 
# Heteroskedasticity can be addressed by using robust standard errors or transforming variables.
X_train_with_const= sm.add_constant(X_train)
white_test = het_white(residuals, X_train_with_const)
print("White's test p-value", white_test[1])

# Normality of Residuals (Shapiro-Wilk test)
# Shapiro-Wilk test p-value less than 0.05 indicates non-normality.
# Non-normality can be addressed by transforming variables or using non-parametric methods.
shapiro_test = stats.shapiro(residuals)
print('Shapiro-Wilk test p-value', shapiro_test[1])

# Multicollinearity check
# VIF greater than 10 indicates multicollinearity.
# Multicollinearity can be addressed by removing highly correlated variables, using ridge regressb or other dimensionality reduction techniques.
vif_data = pd.DataFrame()
vif_data["Variable"] = X_train.columns
vif_data["VIF"] = [variance_inflation_factor(X_train.values, i) for i in range(X_train.shape[1])]
print("Variance Inflation Factor (VIF):\n", vif_data)

# e) It is important that the model coefficients are constant because they represent the intercept and slope of the linear relationship 
# between the variables. If these coefficients are not constant, it implies that the relationship between independent and depent variables 
# is not consistent across the entire dataset, which undermines the validity and interpretability of the model.

# Constant coefficients ensure that the interpretation of the model remains consistent. For example, if  represents the effect of X on Y, 
# a constant  implies that this effect remains the same across all observations. A model with constant coefficients provides stable predictions. 
# Inconsistent coefficients can lead to erratic predictions, making the model unreliable for forecasting.

# To test whether the coefficients are constant, various diagnostics techniques can be employed, including:

# Visual Inspection: Plotting the residuals against the predicted values or against the independent variables can reveal patterns or trends, 
# indicating non-constant variance or relationships.

# Statistical Tests: Tests such as the Breusch-Pagan test, Goldfeld-Quandt test, or White test can formally assess the homogeneity of variance or 
# the constancy of coefficients.

# Time Series Analysis: For time series data, techniques such as residual autocorrelation plots or stability tests like the Chow test can be used 
# to examine the constancy of coefficients over time.

# If the constancy of coefficients assumption fails, meaning that the coefficients are not constant, several steps can be taken:

# Model Re-specification: Re-evaluate the model specification, including adding new variables, transforming variables, or considering alternative 
# functional forms to better capture the underlying relationships in the data.

# Data Transformation: Applying transformations to the variables (e.g., logarithmic or power transformations) might help stabilize the relationships 
# and make the coefficients more constant.

# Robust Regression Techniques: Using robust regression methods that are less sensitive to outliers or heteroscedasticity, such as robust standard 
# errors or weighted least squares, can mitigate the impact of non-constant coefficients.

# f) The notation ~ iid N(0,^2)  indicates that the error term follows a normal (Gaussian) distribution with a mean of 0 and constant variance.
# IID: The "IID" part means that the error terms are independent and identically distributed. Independence implies that the errors for different 
# observations are not correlated with each other. Identically distributed means that each error term has the same probability distribution, in this
# case, a normal distribution.

# Normal Distribution: The error terms are assumed to follow a normal distribution. This assumption is essential because many statistical methods 
# and tests rely on the assumption of normality, such as hypothesis tests and confidence intervals. Additionally, the normal distribution is convenient 
# because it is well-understood and characterized by its mean and variance.

# Mean and Variance: The mean of the error term is specified as 0, which implies that, on average, the errors do not systematically overestimate or 
# underestimate the true values of the dependent. constant variance indicates that the variability of the errors is consistent across all observations.

# The assumption of ~ iid N(0,^2)  matters because it underpins many of the statistical properties used in regression analysis. Violations of this
# assumption can lead to biased estimates, incorrect standard errors, and misleading hypothesis tests.
